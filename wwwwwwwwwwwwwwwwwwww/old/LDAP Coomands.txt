1. Levantar OpenLDAP con docker run
Versión mínima correcta
docker run --detach \
  --name openldap \
  --hostname ldap.lab.local \
  -p 389:3890 \
  -p 636:6360 \
  -e OPENLDAP_BOOTSTRAP_ORGANIZATION="Lab Org" \
  -e OPENLDAP_BOOTSTRAP_SUFFIX="dc=lab,dc=local" \
  osixia/openldap:latest

Esto sigue el esquema base documentado por la imagen.

Versión persistente recomendada
docker run --detach \
  --name openldap \
  --hostname ldap.lab.local \
  -p 389:3890 \
  -p 636:6360 \
  -e OPENLDAP_BOOTSTRAP_ORGANIZATION="Lab Org" \
  -e OPENLDAP_BOOTSTRAP_SUFFIX="dc=lab,dc=local" \
  -v openldap-conf:/etc/openldap/slapd.d \
  -v openldap-data:/var/lib/openldap/openldap-data \
  -v openldap-backups:/var/lib/openldap/openldap-backups \
  osixia/openldap:latest

La documentación de la imagen recomienda montar esos directorios para persistencia.

2. Entrar al contenedor
docker exec -it openldap /bin/bash

Si quieres entrar como root:

docker exec -it -u root openldap /bin/bash
3. Verificar que está corriendo

En vez de depender solo de ps aux | grep slapd, primero revisa:

docker ps
docker logs openldap

Y dentro del contenedor, si quieres:

ps aux | grep slapd

También puedes subir el nivel de logs con:

docker run -e OPENLDAP_DEBUG_LEVEL=-1 osixia/openldap

o pasar opciones a slapd. Eso está soportado por la imagen.

4. Iniciar OpenLDAP manualmente

En esta imagen normalmente no debes arrancarlo a mano como en Bitnami con:

/opt/bitnami/scripts/openldap/run.sh

Eso es de Bitnami, no de osixia.

En osixia/openldap, el proceso se arranca automáticamente al levantar el contenedor.
Si quieres pasar argumentos a slapd, se hace así:

docker run osixia/openldap -- -d -1

Eso también viene documentado en la imagen.

5. Rutas internas correctas

Tus rutas viejas:

/bitnami/openldap/slapd.d/...

Con osixia/openldap ya no aplican.

Rutas correctas

Configuración:

/etc/openldap/slapd.d

Datos:

/var/lib/openldap/openldap-data

Backups:

/var/lib/openldap/openldap-backups

Eso lo define la propia imagen.

6. Ver archivo de configuración LDAP

En lugar de:

cat /bitnami/openldap/slapd.d/cn\=config/olcDatabase\=\{2\}mdb.ldif

usa algo como:

cat /etc/openldap/slapd.d/cn=config/olcDatabase={1}mdb.ldif

o lista primero para ver el índice exacto:

ls /etc/openldap/slapd.d/cn=config

Porque el número {1}, {2}, etc. puede variar.

7. Buscar todo bajo tu Base DN

Si usas:

sufijo: dc=lab,dc=local

el comando actualizado sería:

ldapsearch -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -b "dc=lab,dc=local" \
  -s sub "(objectClass=*)"

Notas:

con esta imagen, por defecto el root DN de datos es cn=admin,${OPENLDAP_BOOTSTRAP_SUFFIX}.
desde fuera del contenedor usas ldap://localhost:389 porque mapeaste 389:3890
8. Ver solo usuarios
ldapsearch -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -b "dc=lab,dc=local" \
  -s sub "(objectClass=inetOrgPerson)"
9. Ver un usuario específico
ldapsearch -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -b "dc=lab,dc=local" \
  "(uid=usuario.desarrollo1)"
10. Cambiar contraseña de un usuario
ldappasswd -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -S "uid=usuario.desarrollo1,ou=Desarrollo,dc=lab,dc=local"
11. Generar hash de password

La imagen trae helper oficial:

Generar password aleatorio
docker run --rm osixia/openldap --cmd -- openldap-ctl password generate
Hashear un password
docker run --rm osixia/openldap --cmd -- openldap-ctl password hash tu_password

Eso está documentado por la imagen.

Si quieres seguir usando slappasswd, también puedes, pero este helper ya viene preparado.

12. Agregar objetos con LDIF
Copiar el archivo al contenedor
docker cp add_items.ldif openldap:/tmp/add_items.ldif
Ejecutar ldapadd
ldapadd -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -f add_items.ldif

O dentro del contenedor:

ldapadd -x -H ldap://localhost:3890 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -f /tmp/add_items.ldif

Si lo ejecutas dentro del contenedor, usa el puerto interno 3890.
Si lo ejecutas fuera, usa el host mapeado 389.

13. Modificar LDAP con LDIF
ldapmodify -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  -f groupmod.ldif
14. Borrar un objeto
ldapdelete -x -H ldap://localhost:389 \
  -D "cn=admin,dc=lab,dc=local" \
  -W \
  "uid=portainer_bind.svc,ou=users,dc=lab,dc=local"

changetype: modify

17. Lo más importante que cambió
Antes, con Bitnami
variables tipo LDAP_ROOT, LDAP_ADMIN_USERNAME
rutas /bitnami/openldap/...
puertos internos 1389, 1636
script /opt/bitnami/scripts/openldap/run.sh
Ahora, con osixia
variables OPENLDAP_BOOTSTRAP_ORGANIZATION, OPENLDAP_BOOTSTRAP_SUFFIX
rutas /etc/openldap/... y /var/lib/openldap/...
puertos internos 3890, 6360
arranque automático del servicio
helpers openldap-ctl para passwords/backups
